Building a Visual Video Editor for the Web

Using popcorn.js to change how we use video.

Created by Mike Nolan / @TheRogueNolski

Hello!

Hi, I'm Mike
I'm a software developer at mozilla
I work on a product called Air Mozilla which is a video streaming site akin to youtube but most recently I have been working on creating an in browser solution for editing uploaded videos
Most of my work has stemmed from Popcorn Maker, a project built by the Mozilla Foundation Webmaker team. During my time on this project I've created a fork which contains strictly the front end components for the editor and allows users to hook up javascript events for loading and saving content through the editor.
I was then approached by Ben Moskowitz from Mozilla Foundation and introduced to SJ Klein at WMF They introduced me to a lot of the great video initiatives going on within the wikimedia communities
So now I'm here, at Wikimania, with all of you, doing this talk. Pitching this editor and video content and how it can change how we think of Open Knowledge. So before I go on about the tech I have built I want to try and show how important video has been in the evolution of the web and how we consume information.

200 petabytes of video played on Vimeo in 2012

as I'm sure you've all heard video on the web has been increasing an incredible amount. In 2012 Vimeo reported that users played over 200 petabytes of video.
Videos on the internet are no longer simply cat videos and people being kicked in the testicles either.
The web is full of great things. Things that help you learn and share knowledge. 7000 hours of video are uploaded to youtube daily. Videos uploaded that were tagged Obama or Romney were viewed 2.7 BILLION times during the 2012 election cycle. That's 3x the views of Gangnam style
As a student I probably spend the same amount of time consuming free educational video content as I do listening to a professor re-iterate the same things for tens of thousands of dollars per year.
So that's my case for why video is important for wikimedia and such. So I've been working on this video video editor to allow users of air mozilla to modify and remix uploaded content. What does it look like? well, it looks something like this. The front end of the editor is this modular javascript library that you can just bower install and then we are implementing a back end message queue powered by ffmpeg that converts the editors output (popcorn.js json) into a flat rendered video file.
